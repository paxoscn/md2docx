# Note Strategy 示例文档

这个文档展示了如何使用 note 类型的代码块。

## 示例 1：基本的 Note

```note
重要提示
这是一个非常重要的注意事项。
请务必仔细阅读并遵守相关规定。
```

## 示例 2：使用 Tip 别名

```tip
专业建议
在提交代码之前，请确保：
1. 所有测试都通过
2. 代码已经过 lint 检查
3. 文档已更新
```

## 示例 3：使用 Hint 别名

```hint
小提示
使用 Ctrl+Shift+P (Windows/Linux) 或 Cmd+Shift+P (Mac) 可以快速打开命令面板。
```

## 示例 4：安全警告

```note
安全警告
永远不要在代码中硬编码敏感信息，如：
- API 密钥
- 数据库密码
- 私钥文件
请使用环境变量或密钥管理服务。
```

## 示例 5：性能提示

```tip
性能优化建议
对于大型数据集的处理，考虑使用以下策略：
- 批量处理而非逐条处理
- 使用索引加速查询
- 实现缓存机制
- 异步处理非关键任务
```

## 示例 6：单行提示

```hint
记得定期备份你的工作！
```

## 示例 7：最佳实践

```note
代码审查最佳实践
1. 保持审查范围小而集中
2. 提供建设性的反馈
3. 关注代码质量和可维护性
4. 及时响应审查请求
5. 学习他人的优秀实践
```

## 示例 8：调试技巧

```tip
调试技巧
遇到难以定位的 bug 时：
- 使用二分法缩小问题范围
- 添加详细的日志输出
- 使用调试器逐步执行
- 尝试重现最小化的测试用例
- 向同事寻求帮助
```

## 示例 9：文档编写建议

```hint
好的文档应该包含：
- 清晰的目的说明
- 使用示例
- API 参考
- 常见问题解答
- 故障排除指南
```

## 示例 10：Git 工作流提示

```note
Git 提交信息规范
使用清晰的提交信息格式：
- feat: 新功能
- fix: 错误修复
- docs: 文档更新
- style: 代码格式调整
- refactor: 代码重构
- test: 测试相关
- chore: 构建/工具相关
```
