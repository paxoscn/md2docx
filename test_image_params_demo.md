# 图片尺寸参数演示

本文档演示如何使用URL参数控制图片尺寸。

## 1. 基本用法

### 1.1 指定宽度和高度

使用 `?width=100&height=50` 参数可以同时指定图片的宽度和高度：

```markdown
![示例图片](img/example.png?width=100&height=50)
```

### 1.2 只指定宽度

使用 `?width=200` 参数只指定宽度，高度将使用默认值：

```markdown
![示例图片](img/example.png?width=200)
```

### 1.3 只指定高度

使用 `?height=150` 参数只指定高度，宽度将使用默认值：

```markdown
![示例图片](img/example.png?height=150)
```

## 2. 实际应用场景

### 2.1 缩略图

对于需要显示为缩略图的图片，可以设置较小的尺寸：

```markdown
![缩略图](img/thumbnail.png?width=80&height=80)
```

### 2.2 横幅图片

对于横幅类型的图片，可以设置较宽的尺寸：

```markdown
![横幅](img/banner.png?width=600&height=100)
```

### 2.3 图标

对于图标类型的图片，可以设置统一的小尺寸：

```markdown
![图标](img/icon.png?width=32&height=32)
```

## 3. 参数说明

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| width | 整数 | 图片宽度（像素） | width=100 |
| height | 整数 | 图片高度（像素） | height=50 |

## 4. 注意事项

1. **参数格式**：参数必须以 `?` 开始，多个参数用 `&` 连接
2. **参数值**：必须是有效的正整数
3. **默认值**：未指定的维度将使用配置文件中的默认值
4. **本地图片**：此功能主要用于本地图片文件
5. **向后兼容**：不带参数的图片URL继续正常工作

## 5. 技术实现

此功能通过以下步骤实现：

1. **解析阶段**：Parser解析Markdown时提取URL中的查询参数
2. **存储阶段**：将宽度和高度信息存储在AST的Image节点中
3. **生成阶段**：Generator根据参数值调整图片尺寸

## 6. 示例对比

### 不使用参数（默认尺寸）

```markdown
![默认尺寸](img/demo.png)
```

### 使用参数（自定义尺寸）

```markdown
![自定义尺寸](img/demo.png?width=150&height=100)
```

## 总结

通过URL参数控制图片尺寸的功能，使得Markdown文档可以更灵活地控制图片的显示效果，而无需修改原始图片文件或使用复杂的HTML标签。
